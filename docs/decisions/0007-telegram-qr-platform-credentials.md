# ADR-0007：Telegram 普通账号仅使用平台凭据二维码登录

> 状态：accepted · 日期：2026-07-12

## 背景

让用户在网页中输入 MTProto API Hash、手机号、验证码、两步验证密码或 Session 会扩大秘密暴露面，且
手机验证码流程在 Telegram 平台上并不稳定。普通账号监听仍需要可恢复的授权会话和常驻进程。

## 决策

平台以环境变量持有统一 API ID/API Hash；用户在网页只扫描短期 QR。QR URL 与登录后 session 都使用
应用加密后存储，API 只将 QR URL 返回给其 owner。专用 QR worker 处理登录生命周期，独立 listener 只读
摄取用户选定的群组/频道；不会发送消息或执行任何外部动作。启用两步验证的账号安全失败，不改回密码输入。

## 后果

- 部署需要两个常驻服务及平台 Telegram 开发者凭据。
- 新页面不再采集用户凭据；旧配置 API 暂时保留，仅用于管理既有 legacy session，后续通过独立迁移计划下线。
- QR worker 重启或二维码过期时用户需要重新扫码，换取不持久化未授权登录状态的安全边界。
