# ADR-0011：工作机会发现采用证据约束提取与确定性匹配

> 状态：accepted · 日期：2026-07-16

## 背景

招聘信息来自高噪音群聊，就业又属于高影响领域。直接让模型生成职位和推荐分数会造成字段
编造、不可解释排序和受保护属性进入决策的风险。仓库已经有受限 pi Agent Runtime、消息级
用量账本和异步分析路径，不应为垂直场景再建立一套运行时。

## 决策

1. 工作机会分析扩展现有受限 pi Agent Runtime，模型只做来源画像、消息分类、字段提取和解释。
2. 每个重要职位字段必须附带来自授权消息上下文的短证据；无证据字段降为 null/unknown。
3. 最终匹配分由 Python 纯领域服务按用户主动声明的职业因素计算，模型不能提交或覆盖分数。
4. 年龄、性别、民族、宗教、婚育、残障和政治倾向不得进入用户档案或匹配输入。招聘原文中的
   显式限制只产生透明度字段和合规警告。
5. 职位写入、owner 校验和去重均由 application/repository 完成，Node Agent 不获得数据库访问。
6. 复用 message 级 UsageLedger 幂等键。普通聊天先经过确定性预筛，避免每条消息触发模型。
7. 去重先使用可解释指纹，再使用本地稀疏结构化特征向量的余弦相似度。该向量不是外部神经
   embedding，不新增 provider 调用；公司、城市或资历冲突时禁止合并。

## 后果

- 职位结果可追溯且排序可测试，模型升级不会改变评分权威。
- 缺失信息会降低展示完整度，但系统不会用行业常识补齐未知字段。
- 当前第二层去重可以容忍轻微文案改写，但不具备通用语义 embedding 的召回能力。未来接入外部
  provider 前必须重新评估消息隐私、模型成本、阈值和故障回退。
- 客户端只能展示后端投影，不能提交 match score 或受保护属性来影响排序。
